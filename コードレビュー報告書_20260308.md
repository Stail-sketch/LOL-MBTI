# LOL-MBTI コードレビュー報告書

**実施日**: 2026-03-08
**分析対象**: LOL-MBTI プロジェクト全体（HTML / CSS / JavaScript / データ）

---

## 1. 分析概要

3つのエージェントを並行起動し、以下の観点でプロジェクトを分析した。

| エージェント | 担当範囲 |
|---|---|
| JS分析 | js/ フォルダのコード品質・アーキテクチャ |
| HTML/CSS分析 | UI構造・スタイル・アクセシビリティ |
| データ分析 | data.js の内容・MBTI診断ロジック・作業履歴 |

## 2. 発見された問題（分析結果）

### 🔴 緊急度：高

| # | 問題 | 場所 | 内容 |
|---|---|---|---|
| 1 | **Firebase APIキー露出** | `js/firebase.js` | APIキーがソースコードに直書き。公開リポジトリなら第三者が悪用可能 |
| 2 | **エラー握りつぶし** | `js/storage.js` | `catch(e){}` でJSON解析エラーを無視。破損データが検出できない |
| 3 | **画像読み込み競合バグ** | `js/export.js` | `onload`/`onerror` が両方発火する可能性。cleanup処理が未実装 |

### 🟡 重要度：高

| # | 問題 | 場所 | 内容 |
|---|---|---|---|
| 4 | **レーダー描画の重複** | `js/export.js` / `js/ui.js` | `drawRadarOnCard()` と `drawRadar()` がほぼ同一ロジックで重複 |
| 5 | **状態リセットコードの重複** | `js/main.js` | `startDiagnosis` / `retryDiagnosis` / `goToTop` に同じ初期化コードが3箇所重複 |
| 6 | **マジックナンバー** | `js/ui.js` / `js/diagnosis.js` | `0, 10, 25`（フェーズオフセット）と `10, 15`（フェーズ問題数）が定数化されていない |
| 7 | **CSSカラー変数の重複** | `style.css` / `privacy.html` | `:root` カスタムプロパティが2ファイルに同一定義 |

### 🟢 低優先度

| # | 問題 | 場所 | 内容 |
|---|---|---|---|
| 8 | **html2canvas 常時読み込み** | `index.html` | 約100KBのライブラリを毎回読み込み。ダウンロード機能使用時のみで十分 |
| 9 | **アクセシビリティ不足** | `index.html` | モーダルに `role="dialog"` なし、SVGに `aria-hidden` なし、ボタンに `aria-label` なし |
| 10 | **未使用CSSセレクタ** | `style.css` | `.qcount-right{}` / `.ext-label-placeholder` / `.axis-row-right{}` が空/未使用 |
| 11 | **タブレット対応なし** | `style.css` | ブレークポイントが 700 / 500 / 420px のみ。768px 台のタブレットで対応が不十分 |

### 📋 その他（今回は対応しない）

| 問題 | 内容 |
|---|---|
| グローバル変数多用 | `main.js` に9つのグローバル変数。状態管理オブジェクト化を推奨するが影響範囲が大きいため保留 |
| チャンピオンスコアの主観性 | 167体のスコアが設計者の主観に依存。コミュニティ検証の仕組みが必要 |
| インラインイベントハンドラ | `onclick=` が30箇所以上。JSファイルへの分離が望ましいが規模が大きいため保留 |
| Google Fonts 読み込み過多 | Cinzel / Noto Serif JP / Rajdhani の3フォントを毎回読み込み |
| 言語データ構造の不統一 | 日本語/英語データが別オブジェクトに分散。`{ja, en}` 形式への統一を推奨 |

## 3. 実施した修正

### 修正① Firebase APIキー露出（緊急）

**変更ファイル**: `js/firebase.js` / `js/firebase-config.js`（新規） / `js/firebase-config.example.js`（新規） / `.gitignore`（新規） / `index.html`

**内容**:
- APIキーを `js/firebase-config.js` に分離し、`.gitignore` で除外
- `js/firebase-config.example.js` をテンプレートとしてリポジトリに追加
- `js/firebase.js` は `FIREBASE_CONFIG` 変数を参照するだけに変更
- `index.html` で `firebase-config.js` を先に読み込むよう修正

> ⚠️ 過去のgit履歴にはAPIキーが残存。Firebase Consoleでドメイン制限を設定することを推奨。

---

### 修正② エラー握りつぶし（緊急）

**変更ファイル**: `js/storage.js`

**変更前**:
```js
} catch(e) {}
```

**変更後**:
```js
} catch(e) {
  console.error('前回結果の読み込みに失敗しました。データを削除します。', e);
  localStorage.removeItem('lol_last_result');
}
```

---

### 修正③ 画像読み込み競合バグ（緊急）

**変更ファイル**: `js/export.js`

**変更前**:
```js
img.onload = r;
img.onerror = r;
```

**変更後**:
```js
const cleanup = () => { img.onload = null; img.onerror = null; };
img.onload = () => { cleanup(); r(); };
img.onerror = () => { cleanup(); console.warn('画像読み込み失敗:', img.src); r(); };
```

---

### 修正④ レーダー描画の重複統一（重要）

**変更ファイル**: `js/ui.js` / `js/export.js`

`drawRadarChart(canvas, norm)` を `ui.js` に共通関数として定義。
`drawRadar()` と `drawRadarOnCard()` はそれぞれ1行のラッパーに変更（約15行の重複コード削除）。

```js
// ui.js
function drawRadarChart(canvas, norm) { /* 共通ロジック */ }
function drawRadar(norm) { drawRadarChart(document.getElementById('radarCanvas'), norm); }

// export.js
function drawRadarOnCard(norm) { drawRadarChart(document.getElementById('ce-radar'), norm); }
```

---

### 修正⑤ 状態リセット重複の解消（重要）

**変更ファイル**: `js/main.js`

`resetState()` を抽出。`retryDiagnosis()` は `startDiagnosis()` の呼び出しに簡略化。

```js
function resetState() {
  selectedLane = null; currentPhase = 1; currentQ = 0; currentPhaseEnd = 10;
  scores = {V:0,I:0,H:0,T:0,A:0,W:0,S:0,D:0};
  laneResultsCache = {}; lastNormalized = null; answerHistory = [];
}
function startDiagnosis() { resetState(); allActiveQuestions = initQuestions(); ... }
function retryDiagnosis() { startDiagnosis(); }  // 重複コードを廃止
```

## 5. 今後の推奨対応

| 優先度 | 内容 |
|---|---|
| 高 | Firebase Console でAPIキーのドメイン制限を設定する |
| 中 | グローバル変数を状態管理オブジェクトにまとめる（影響範囲が大きいため段階的に） |
| 中 | インラインの `onclick=` をJSファイルのイベントリスナーに移す |
| 低 | Google Fonts の読み込みフォント数を削減する（Rajdhani or Cinzel のどちらかを検討） |
| 低 | チャンピオンスコアの検証（コミュニティやプロプレイヤーからのフィードバック収集） |
| 低 | 言語データ構造を `{ja: "...", en: "..."}` 形式に統一する |
