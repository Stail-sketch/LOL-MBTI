# LOL-MBTI「サモナーの素質診断」サイト分析レポート

## 1. ページ構成

### 全体概要
- **単一ファイル構成**: `index.html`（2,699行, 296KB）にHTML・CSS・JS全てを統合したSPA
- **6画面構成**: スタート → 質問(10問) → ロール選択 → 質問(25問/40問) → ローディング → 結果
- **段階的診断**: 10問→25問→40問と段階的に深掘りし、回答数に応じて結果の詳細度が増す

### 画面フロー
```
スタート画面 → 質問(10問) → ロール選択(TOP/JG/MID/ADC/SUP/ANY)
    → ローディング → 結果表示
    → 「さらに答える」→ 質問(25問目まで) → 結果更新
    → 「さらに答える」→ 質問(40問目まで) → 完全結果表示
```

### 診断ロジック
- **8軸評価**: V(外向性), I(直感力), H(共感性), T(計画性), A(積極性), W(リスク志向), S(協調性), D(忍耐力)
- **167体のチャンピオン**に8軸スコアを手動設定
- **ユークリッド距離**でユーザーとチャンピオンの類似度を算出
- **8つのサモナータイプ**（ruler, commander, strategist, intuitive, guardian, berserker, empath, adventurer）

### 主要機能
- 日本語/英語切り替え
- 結果カード画像ダウンロード（html2canvas）
- X(Twitter)シェア
- Firebaseによるコミュニティランキング
- レーダーチャート（Canvas描画）
- 前回結果のlocalStorage保存・復元

## 2. 現在の問題点

### A. 構造・保守性
| 問題 | 詳細 |
|------|------|
| **巨大な単一ファイル** | 2,699行のHTMLにCSS(~450行)・JS(~1,860行)・データ(167体分)が混在。保守困難 |
| **コード分離なし** | ロジック・データ・UIが分離されておらず、変更時の影響範囲が広い |
| **インラインスタイル散在** | `style="display:none"` 等がHTML内に散在 |

### B. パフォーマンス
| 問題 | 詳細 |
|------|------|
| **初期ロード重い** | 296KBのHTMLを一括ロード（167体分のデータ含む） |
| **Firebase大量取得** | ランキング表示時に最新1,000件を毎回取得（キャッシュなし） |
| **外部CDN依存** | スプラッシュアート画像を`cdn.communitydragon.org`から都度取得 |
| **DOM毎回再生成** | `renderQuestion()`で質問表示のたびにDOM全書き換え |

### C. SEO・OGP
| 問題 | 詳細 |
|------|------|
| **og:url 欠落** | 正規URLが未設定 |
| **og:locale 欠落** | 多言語対応にも関わらずlocale情報なし |
| **動的OGPの限界** | JSでOGP画像URLを書き換えるが、クローラーがJSを実行しない場合に失敗 |
| **結果個別OGPなし** | シェア時に「どのチャンピオンが出たか」がOGPに反映されない |
| **ファビコン未設定** | `<link rel="icon">` がない |

### D. アクセシビリティ
| 問題 | 詳細 |
|------|------|
| **セマンティクス不足** | `<main>`, `<nav>`, `<article>` タグ未使用 |
| **ARIA属性欠落** | ロールカード等にrole/aria属性なし |
| **キーボード操作** | `.role-card`がdivのonclickのみでキーボード非対応 |
| **進捗バー非通知** | スクリーンリーダーに進捗状況が伝わらない |

### E. UX・診断体験
| 問題 | 詳細 |
|------|------|
| **質問の単調さ** | 40問すべてA/B二択で、途中で飽きやすい |
| **フェーズ切替の分かりにくさ** | 10問→ロール選択→結果→「さらに答える」のフローが直感的でない |
| **戻るボタンの制約** | フェーズ1では戻るボタンが無効化されている |
| **ローディング演出の意味** | 実際の計算は一瞬だが演出で待たせている（演出自体は良いが長すぎる可能性） |
| **結果の再現性** | 同じ回答でも表示が変わらないことの説明がない |

## 3. 改善提案

### 優先度：高（UX・診断体験の向上）

#### 3-1. 質問体験の改善
- **質問にLoLテーマを強化**: 現在の質問は一般的な性格診断寄り。LoLのシチュエーション（「チームメイトがfeedしている時…」等）を増やす
- **質問バリエーション**: スライダー形式や3択以上の質問を混ぜて単調さを軽減
- **進捗の可視化強化**: 「残りX問」「あとY問で詳細結果が解放」等の表示

#### 3-2. 結果画面の充実
- **チャンピオンとの比較レーダー**: ユーザーのレーダーチャートとチャンピオンのレーダーチャートを重ねて表示
- **マッチ率の解釈説明**: 「98%」が何を意味するかの説明
- **他チャンピオンとの比較**: TOP5だけでなくワースト（天敵）も視覚的に見せる
- **タイプ別の割合表示**: 「あなたと同じタイプは全体のX%」のような統計情報

#### 3-3. シェア体験の強化
- **結果画像にチャンピオン名・マッチ率を含む**: SNSで見た人が興味を持ちやすい
- **シェア文言のカスタマイズ**: テンプレート複数パターン用意

### 優先度：中（技術的改善）

#### 3-4. コード分割・モジュール化
- `data/champions.js` - チャンピオンデータ
- `data/questions.js` - 質問データ
- `data/types.js` - サモナータイプデータ
- `logic/diagnosis.js` - 診断計算ロジック
- `ui/screens.js` - 画面遷移管理
- Vite等のバンドラー導入

#### 3-5. パフォーマンス改善
- スプラッシュアート画像のlazy loading
- Firebaseデータのキャッシュ（localStorage + TTL）
- チャンピオンデータの遅延読み込み

#### 3-6. SEO・OGP修正
- `og:url`, `og:locale` の追加
- ファビコン設定
- 結果シェア時の動的OGP（サーバーサイド or OGP画像生成サービス活用）

### 優先度：低（品質向上）

#### 3-7. アクセシビリティ
- セマンティックHTML（`<main>`, `<nav>`, `<article>`）
- ARIA属性の追加
- キーボードナビゲーション対応

#### 3-8. その他
- PWA対応（オフライン診断可能に）
- 診断履歴機能（過去の結果を一覧表示）
- チャンピオンフィルター（ロール×タイプで絞り込み）

---

## まとめ

このサイトは**診断ロジック・ビジュアル・多言語対応**が充実した完成度の高い診断サイトです。最大の課題は**単一ファイル構成による保守性の低さ**と、**診断体験のLoLらしさの不足**です。技術的なリファクタリングよりも、まず**UX面（質問のLoLテーマ強化・結果画面の充実・シェア体験）**を改善することで、ユーザー満足度とバイラル効果を高められると考えます。
