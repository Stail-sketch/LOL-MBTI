# ポップアップ（インタースティシャル）広告について

## 収益面の比較

| 形式 | CPM目安 | 1訪問あたりの収益 | 必要訪問者（第2目標） |
|------|--------|---------------|-------------------|
| バナー3枚（従来案） | ¥200 | ¥0.273 | 834人/日 |
| **フルスクリーン2回（採用案）** | **¥400** | **¥0.364** | **626人/日** |

インタースティシャルはCPMが高く、**バナーより少ない訪問者で稼げる。**
（表示回数は少ないが単価が高いため）

---

## AdSenseポリシー上の可否

| ケース | 可否 | 理由 |
|--------|------|------|
| 診断開始ボタン押下 → 広告 → 診断へ | **◯ OK** | 自然な画面遷移のタイミング |
| 結果表示直前 → 広告 → 結果へ | **◯ OK** | 同上 |
| コンテンツ閲覧中に突然出現 | **× NG** | AdSenseポリシー違反 |
| X ボタン即時表示（すぐ消せる） | **◯ OK** | 必須要件を満たす |
| X ボタンを5秒後に表示 | **× NG** | ポリシー違反 |

**重要：X ボタンは表示直後から押せる状態にする必要がある。**

---

## 最終仕様

| 項目 | 内容 |
|------|------|
| 表示タイミング | ①診断開始ボタン押下時、②結果表示直前 |
| 表示形式 | 黒背景フルスクリーンオーバーレイ |
| 広告サイズ | 300x250（レクタングル、中央配置） |
| クローズ | 右上の×ボタンで即時クローズ（×は最初から表示） |
| 自動クローズ | なし（×押すまで表示し続ける） |
| AdSense対応 | ◯（ポリシー違反なし） |

## 実装内容（index.html）

### 追加したもの
1. フルスクリーンオーバーレイのCSS（`#ad-overlay` スタイル）
2. オーバーレイHTML（`<body>` 直後）
3. JS関数：`showAdOverlay(callback)` / `closeAdOverlay()`

### 変更したもの
- `startDiagnosis()` → `showAdOverlay()` 経由で質問画面へ
- `showResult()` → `showAdOverlay()` 経由で結果画面へ（内部処理は `_showResultInner()` に分離）

### AdSense承認後の差し替え箇所
`index.html` の広告オーバーレイHTML内、コメントアウトされた `<ins>` タグを有効化し、`ca-pub-XXXXX` と `data-ad-slot` を設定する。
